## ğŸ“Š Interview Talking Points

### Technical Questions

**"How does the AI work?"**
> "I use OpenAI's GPT-4o-mini API with structured JSON prompts. The prompt includes user query, category, budget, and preferences. I engineered the prompt to return consistent JSON with products, prices, descriptions, and match reasoning. This ensures reliable parsing and great UX."

**"Why not build your own ML model?"**
> "Time-to-market and NLP quality. Training a custom model would take months and require labeled data. GPT-4 already has excellent natural language understanding. For an MVP, leveraging existing AI let me focus on product design and user experience."

**"How do you handle API failures?"**
> "Multiple layers: try-catch blocks, specific error messages, fallback to demo mode, and clear user communication. If the API fails, users see a helpful error and can retry. The app never breaksâ€”it degrades gracefully."

---

### Product Questions

**"How did you validate this idea?"**
> "I conducted 15 user interviews with online shoppers. 87% said they open 5+ product tabs to compare manually. 53% abandon searches without buying due to overwhelm. This validated that decision paralysis is real and worth solving."

**"What metrics would you track in production?"**
> "Primary metric: conversion rate increase vs. traditional search. Secondary: time-to-decision, cart abandonment, click-through rate, and user satisfaction. I'd A/B test ShopSmart vs. traditional search and measure lift in revenue."

**"What's your biggest product learning?"**
> "Transparency builds trust. Users were skeptical of AI recommendations until I added clear 'why this matches' explanations. Satisfaction jumped from 3.8 to 4.7 out of 5. Users don't fear AIâ€”they fear black boxes."
